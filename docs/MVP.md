# MVP Definition

## Общая информация

**Цель MVP:** Создать полную мультиагентную систему для разработки мобильного приложения на React Native.

**Первый проект:** Мобильное приложение на React Native (состав продукта определяется founder'ом)

---

## Состав агентов

**Полный набор из 7 ролей:**

1. 🎯 **Product Manager** — сбор требований, PRD, бэклог
2. 📊 **Analyst** — декомпозиция фич на задачи
3. 🏗️ **Architect** — system design, ADR
4. 🎨 **Designer** — дизайн-система, UI-макеты
5. 💻 **Developer** — написание кода
6. 🧪 **QA** — тестирование, баг-репорты
7. 📝 **Tech Writer** — документация

---

## Функционал MVP

### Обязательно:

- [ ] Приём визии от founder'а через Telegram/CLI
- [ ] Создание PRD на основе диалога
- [ ] Генерация бэклога в GitHub Issues/Projects
- [ ] Декомпозиция фич на задачи
- [ ] Создание ADR и System Design
- [ ] Создание дизайн-системы (code-first)
- [ ] Написание кода по задачам
- [ ] Unit-тесты (Developer)
- [ ] E2E/Интеграционные тесты (QA)
- [ ] Code review (Developer на другой модели)
- [ ] Баг-репорты в GitHub Issues
- [ ] Верификация исправлений
- [ ] Сборка приложения

### НЕ входит в MVP:

- ❌ Деплой на прод (делает founder вручную)
- ❌ CI/CD пайплайны
- ❌ Мониторинг и алертинг
- ❌ Автоматические релизы

---

## Первый проект

**Тип:** Мобильное приложение на React Native

**Стек:**
- Frontend: React Native
- Backend: Node.js (если нужен)
- State: Redux / Zustand
- Navigation: React Navigation

**Артефакты:**
- PRD
- ADR и System Design
- Дизайн-система
- Исходный код
- Тесты
- Документация

---

## Архитектура приложения

Архитектор (Architect agent) проектирует:
- Структуру приложения
- Навигацию
- Слои (presentation, business, data)
- Интеграции (если есть)
- Хранение данных

---

## Критерии успеха MVP

1. ✅ Founder может описать идею → система создаёт PRD
2. ✅ PRD превращается в задачи в GitHub
3. ✅ Задачи выполняются агентами → появляется код
4. ✅ Код тестируется и ревьюится
5. ✅ Приложение собирается
6. ✅ Founder может видеть все артефакты и контролировать процесс

---

## Риски MVP

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Качество кода агентов | Средняя | Code review + итерации |
| Сложность координации агентов | Средняя | Поэтапный запуск ролей |
| Токен-лимиты LLM | Низкая | OpenRouter, разные модели |
| Ошибки в архитектуре | Средняя | Ревью founder'ом ADR |

---

## Дорожная карта MVP

### Фаза 1: Foundation
- Настроить CrewAI окружение
- Подключить OpenRouter
- Создать базового агента PM
- Интеграция с GitHub API

### Фаза 2: Core Agents
- Реализовать PM → PRD pipeline
- Реализовать Analyst → задачи
- Реализовать Architect → ADR/System Design

### Фаза 3: Development
- Реализовать Designer → дизайн-система
- Реализовать Developer → код
- Реализовать QA → тесты

### Фаза 4: Integration
- Связать всех агентов в pipeline
- Telegram/CLI интерфейс
- Тестовый проект end-to-end

### Фаза 5: Polish
- Документация системы
- Обработка ошибок
- Оптимизация
